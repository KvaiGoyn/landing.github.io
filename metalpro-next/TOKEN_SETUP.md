# Руководство по настройке токенов и API ключей для сайта Metalpro Next

## Введение

Этот документ содержит полный список всех токенов, API ключей и переменных окружения, необходимых для работы сайта "Стиль Мастер" (Metalpro Next.js приложение). Без правильной настройки этих параметров некоторые функции сайта могут не работать, включая отправку форм, отображение карт и уведомления.

Все переменные окружения должны быть добавлены в файл `.env.local` для локальной разработки или в настройки окружения вашего хостинг-провайдера (Timeweb, Vercel и т.д.) для продакшена.

---

## 📋 Полный список токенов

### 🗺️ 1. Яндекс Карты API
**Переменная:** `NEXT_PUBLIC_YANDEX_MAPS_API_KEY`  
**Назначение:** Отображение интерактивной карты на странице контактов  
**Обязательность:** ✅ Обязательно  
**Пример значения:** `01a47426-71fa-4a26-9ae0-3796e2e33465`  
**Где получить:**
1. Перейдите на [Яндекс.Облако](https://cloud.yandex.ru/)
2. Создайте аккаунт или войдите в существующий
3. В консоли управления выберите "Сервисы" → "API-ключи"
4. Нажмите "Создать API-ключ"
5. Выберите тип "JavaScript API и HTTP Геокодер"
6. Скопируйте сгенерированный ключ
7. Добавьте ключ в переменную окружения

**Примечание:** Ключ публичный (начинается с `NEXT_PUBLIC_`), поэтому он будет виден в исходном коде страницы. Это нормально для Яндекс Карт.

---

### 🤖 2. Telegram Bot Token
**Переменная:** `TELEGRAM_BOT_TOKEN`  
**Назначение:** Отправка уведомлений о новых заявках в Telegram  
**Обязательность:** ✅ Обязательно  
**Пример значения:** `8657525858:AAH7ttt9XggtJLPP2rU1IR3CQLmnv30dD4M`  
**Где получить:**
1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Укажите имя бота (например, "Metalpro Notifications")
4. Укажите username бота (должен заканчиваться на `bot`, например `metalpro_notifications_bot`)
5. Скопируйте токен, который выдаст BotFather (формат: `число:буквенно-цифровая_строка`)
6. Добавьте токен в переменную окружения

**Примечание:** Токен является секретным, не публикуйте его в открытых репозиториях.

---

### 💬 3. Telegram Chat ID
**Переменная:** `TELEGRAM_CHAT_ID`  
**Назначение:** ID чата или пользователя, куда будут отправляться уведомления  
**Обязательность:** ✅ Обязательно  
**Пример значения:** `5137096691`  
**Где получить:**
1. Откройте Telegram и найдите бота [@userinfobot](https://t.me/userinfobot)
2. Отправьте ему любое сообщение
3. Бот ответит с вашим ID пользователя (поле `Id`)
4. Скопируйте этот числовой ID
5. Добавьте ID в переменную окружения

**Альтернативный способ:**
- Создайте группу в Telegram и добавьте туда вашего бота
- Отправьте любое сообщение в группу
- Перейдите по ссылке `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
- Найдите `chat.id` в ответе API

### 📧 4. SMTP Яндекс Почта (для отправки email)
**Переменные:**
- `SMTP_USER` - email Яндекс аккаунта
- `SMTP_PASS` - пароль приложения
- `SMTP_HOST` - хост SMTP сервера (по умолчанию: `smtp.yandex.ru`)
- `SMTP_PORT` - порт (по умолчанию: `465`)
- `SMTP_FROM` - email отправителя (по умолчанию: значение `SMTP_USER`)
- `EMAIL_RECIPIENT` - email получателя уведомлений (по умолчанию: `nezabut123@gmail.com`)

**Назначение:** Отправка email-уведомлений о новых заявках  
**Обязательность:** ✅ Обязательно (SMTP_USER и SMTP_PASS)  
**Пример значения SMTP_PASS:** `gbumacfxbztjxyzs` (пароль приложения, не пароль от почты!)  
**Где получить:**
1. Войдите в ваш Яндекс аккаунт на [id.yandex.ru](https://id.yandex.ru)
2. Перейдите в раздел "Безопасность" → "Пароли приложений"
3. Нажмите "Создать новый пароль"
4. Укажите название (например, "Metalpro Site SMTP")
5. Скопируйте сгенерированный пароль (это и будет `SMTP_PASS`)
6. В качестве `SMTP_USER` используйте ваш полный email Яндекс (например, `ваш-логин@yandex.ru`)

**Примечания:**
- Используйте именно пароль приложения, а не основной пароль от аккаунта
- Порт 465 использует SSL-шифрование
- Убедитесь, что в настройках Яндекс.Почты разрешена отправка писем через SMTP

### 📝 5. FormSubmit Recipient Email
**Переменная:** `FORM_SUBMIT_RECIPIENT_EMAIL`  
**Назначение:** Email для получения заявок через FormSubmit (резервный вариант)  
**Обязательность:** ❌ Опционально  
**Пример значения:** `nezabut123@gmail.com`  
**Где получить:** Укажите любой email, на который должны приходить уведомления о заявках.

**Примечание:** Эта переменная используется как резервный получатель, если не настроен SMTP.

### 📊 6. Аналитика (опционально)
**Переменные:**
- `NEXT_PUBLIC_GOOGLE_ANALYTICS_ID` - ID Google Analytics
- `NEXT_PUBLIC_YANDEX_METRICA_ID` - ID Яндекс.Метрики

**Назначение:** Сбор статистики посещений сайта  
**Обязательность:** ❌ Опционально  
**Пример значения GA:** `UA-XXXXXXXXX-X` или `G-XXXXXXXXXX`  
**Пример значения Метрики:** `XXXXXXXX` (числовой ID)  
**Где получить:**

**Google Analytics:**
1. Создайте аккаунт на [analytics.google.com](https://analytics.google.com)
2. Создайте ресурс (property) для вашего сайта
3. В настройках ресурса найдите "Идентификатор отслеживания" (Tracking ID)
4. Скопируйте ID в формате `UA-XXXXXXXXX-X` или `G-XXXXXXXXXX`

**Яндекс.Метрика:**
1. Создайте счетчик на [metrika.yandex.ru](https://metrika.yandex.ru)
2. После создания счетчика перейдите в настройки
3. Найдите "Номер счетчика" (числовой ID)
4. Скопируйте этот номер

**Примечание:** Для работы аналитики необходимо также добавить соответствующие скрипты в код. В текущей версии сайта интеграция с аналитикой не реализована, но переменные зарезервированы.

### ⚙️ 7. Настройки сборки и деплоя
**Переменные:**
- `NEXT_PUBLIC_BASE_PATH` - базовый путь при деплое в поддиректорию
- `NEXT_EXPORT` - флаг статического экспорта (`true`/`false`)
- `ANALYZE` - флаг анализа бандла (`true`/`false`)

**Назначение:** Конфигурация сборки Next.js приложения  
**Обязательность:** ❌ Опционально  
**Пример значения BASE_PATH:** `/metalpro-next` (если сайт размещен в поддиректории)  
**Где получить:** Определяется вашей конфигурацией деплоя.

## 🛠️ Практическое руководство по настройке

### Шаг 1: Создание файла .env.local
Для локальной разработки создайте файл `.env.local` в корне проекта (рядом с `.env.example`):

```bash
cd metalpro-next
cp .env.example .env.local
```

### Шаг 2: Заполнение переменных
Откройте `.env.local` в текстовом редакторе и замените все `your_..._here` на реальные значения:

```env
# Яндекс Карты
NEXT_PUBLIC_YANDEX_MAPS_API_KEY=ваш_ключ_яндекс_карт

# Telegram
TELEGRAM_BOT_TOKEN=ваш_токен_бота
TELEGRAM_CHAT_ID=ваш_id_чата

# SMTP Яндекс
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=ваш-email@yandex.ru
SMTP_PASS=ваш_пароль_приложения
SMTP_FROM=ваш-email@yandex.ru
EMAIL_RECIPIENT=email_получателя@gmail.com

# FormSubmit (опционально)
FORM_SUBMIT_RECIPIENT_EMAIL=email_получателя@gmail.com

# Аналитика (опционально)
NEXT_PUBLIC_GOOGLE_ANALYTICS_ID=UA-XXXXXXXXX-X
NEXT_PUBLIC_YANDEX_METRICA_ID=XXXXXXXX
```

### Шаг 3: Настройка на хостинге
Для продакшена добавьте те же переменные в панели управления вашего хостинг-провайдера:

**Timeweb Cloud:**
1. Перейдите в раздел "Приложения"
2. Выберите ваше приложение
3. Откройте "Переменные окружения"
4. Добавьте все переменные из списка выше

**Vercel:**
1. Перейдите в проект в Vercel Dashboard
2. Откройте "Settings" → "Environment Variables"
3. Добавьте переменные

**Другие платформы:** Ищите раздел "Environment Variables", "Config Vars" или аналогичный.

## 🔍 Проверка работоспособности

После настройки всех токенов проверьте работу ключевых функций:

### 1. Проверка Яндекс Карт
1. Запустите проект: `npm run dev`
2. Откройте страницу контактов (`/contact`)
3. Убедитесь, что карта загружается без ошибок в консоли
4. Проверьте, что маркер отображается корректно

### 2. Проверка отправки форм
1. Заполните любую форму на сайте (например, форму обратного звонка)
2. Отправьте форму
3. Проверьте:
   - Происходит ли редирект на страницу "спасибо"
   - Приходит ли email на указанный в `EMAIL_RECIPIENT` адрес
   - Приходит ли уведомление в Telegram

### 3. Проверка Telegram бота
1. Отправьте тестовое сообщение боту в Telegram
2. Убедитесь, что бот активен (статус "online")
3. Проверьте логи приложения на наличие ошибок отправки

### 4. Проверка SMTP
В проекте есть тестовые скрипты для проверки SMTP:
```bash
# Запустите тест SMTP
node test-smtp.js
```

Если тест проходит успешно, SMTP настроен правильно.

## ⚠️ Советы по безопасности

### 1. Защита секретных токенов
- **Никогда не коммитьте** файл `.env.local` в Git
- Убедитесь, что `.env.local` указан в `.gitignore`
- Используйте разные токены для разработки и продакшена
- Регулярно обновляйте токены (особенно SMTP пароли)

### 2. Безопасность Telegram бота
- Не публикуйте `TELEGRAM_BOT_TOKEN` в открытых репозиториях
- Ограничьте доступ к боту только доверенным чатам
- Регулярно проверяйте логи бота на подозрительную активность

### 3. Безопасность SMTP
- Используйте пароли приложения, а не основные пароли аккаунта
- Регулярно обновляйте пароли приложения (раз в 3-6 месяцев)
- Ограничьте отправку писем только с доверенных IP-адресов (если возможно)

### 4. Мониторинг и логирование
- Включите логирование отправки форм в `app/api/formsubmit/route.ts`
- Настройте алерты на failed отправки
- Регулярно проверяйте квоты использования API (Яндекс Карты, Telegram)

## 🆘 Устранение неполадок

### Проблема: Карта не загружается
**Возможные причины:**
1. Неверный API ключ Яндекс Карт
2. Ключ не активирован в Яндекс.Облаке
3. Превышена дневная квота запросов

**Решение:**
1. Проверьте правильность ключа в `.env.local`
2. Убедитесь, что ключ активирован в консоли Яндекс.Облака
3. Проверьте квоты в разделе "Квоты" Яндекс.Облака

### Проблема: Формы не отправляются
**Возможные причины:**
1. Не настроены SMTP переменные
2. Неверные SMTP credentials
3. Telegram токен или chat ID неверны

**Решение:**
1. Запустите `node test-smtp.js` для проверки SMTP
2. Проверьте логи сервера (`npm run dev` в терминале)
3. Убедитесь, что Telegram бот активен и добавлен в чат

### Проблема: Нет уведомлений в Telegram
**Возможные причины:**
1. Неверный `TELEGRAM_BOT_TOKEN`
2. Неверный `TELEGRAM_CHAT_ID`
3. Бот заблокирован или удален

**Решение:**
1. Проверьте токен через API: `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getMe`
2. Убедитесь, что chat ID соответствует вашему пользователю или группе
3. Пересоздайте бота через @BotFather при необходимости

## 📞 Поддержка

Если у вас возникли проблемы с настройкой токенов:

1. **Проверьте логи приложения:** `npm run dev` и посмотрите ошибки в консоли
2. **Используйте тестовые скрипты:** `test-smtp.js`, `test-smtp-debug.js`
3. **Сверьтесь с документацией:** файлы `GETFORM_SETUP.md`, `DEPLOYMENT_REVIEW.md`
4. **Обратитесь к разработчику:** предоставьте подробное описание проблемы и скриншоты ошибок

**Важно:** Никогда не передавайте реальные токены и пароли при обращении за поддержкой. Используйте маскированные значения (например, `SMTP_PASS=***masked***`).

## 🔄 Обновление токенов

Рекомендуется обновлять токены с следующей периодичностью:
- **SMTP пароли приложения:** каждые 6 месяцев
- **API ключи Яндекс:** ежегодно или при смене тарифа
- **Telegram токен:** только при компрометации

При обновлении любого токена:
1. Создайте новый токен в соответствующем сервисе
2. Обновите переменную окружения
3. Перезапустите приложение
4. Проверьте работоспособность
5. Удалите старый токен из сервиса (если возможно)
