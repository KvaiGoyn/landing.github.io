## Общая информация

Данный отчет представляет результаты автоматизированного тестирования статического лендинга, построенного на Next.js. Тестирование проводилось с использованием фреймворка Playwright для обеспечения кросс-браузерной совместимости и проверки ключевых функциональных возможностей.

**Цели тестирования:**
- Проверка корректности навигации и якорных ссылок
- Валидация работы модальных окон
- Тестирование форм ввода и валидации данных
- Проверка адаптивности на различных устройствах
- Обеспечение стабильности пользовательского интерфейса

**Объем тестирования:**
- Всего тестов: 225
- Успешных: 130 (57.8%)
- Упавших: 95 (42.2%)
- Тестовых файлов: 5
- Браузеров/устройств: 6 (Chromium, Firefox, WebKit, Mobile Chrome, Mobile Safari)

## Методология тестирования

### Подход к тестированию
Использован комбинированный подход, включающий:
- **Функциональное тестирование:** проверка работы ключевых функций лендинга
- **Кросс-браузерное тестирование:** запуск тестов в 6 различных окружениях
- **Адаптивное тестирование:** проверка отображения на мобильных устройствах
- **Интеграционное тестирование:** проверка взаимодействия компонентов

### Инструменты и технологии
- **Playwright:** основной фреймворк для автоматизации
- **Node.js:** среда выполнения
- **HTML Reporter:** генерация визуальных отчетов
- **GitHub Actions (потенциально):** для CI/CD интеграции

### Тестовое окружение
- **Локальный сервер:** `npx serve -p 8080`
- **Базовый URL:** `http://localhost:8080`
- **Таймаут теста:** 30 секунд
- **Ретрии:** 0 в локальном режиме, 2 в CI
- **Параллельное выполнение:** включено

## Детализация по категориям тестов

### 1. Навигация и якорные ссылки (`navigation.test.js`)

**Общее описание:** Тесты проверяют корректность навигации по странице, работу якорных ссылок и структуру контента.

**Ключевые проверки:**
- Загрузка главной страницы с корректным заголовком
- Наличие навигационных якорных ссылок
- Корректность переходов по якорям
- Структура основных разделов страницы

**Результаты:**
- Успешных: 30 из 45 тестов
- Основные проблемы: таймауты при ожидании загрузки, отсутствие элементов в DOM

**Пример успешного теста:**
```javascript
test('Главная страница загружается с корректным заголовком', async ({ page }) => {
  await expect(page).toHaveTitle(/МеталлПро|Металлоизделия/);
  await expect(page.locator('body')).toBeVisible();
});
```

### 2. Модальные окна (`modals.test.js`)

**Общее описание:** Тестирование открытия, закрытия и функциональности модальных окон, включая калькулятор стоимости.

**Ключевые проверки:**
- Открытие модальных окон по клику на соответствующие кнопки
- Закрытие по крестику и клавише Escape
- Фокус внутри модального окна (TAB-навигация)
- Одновременное открытие только одного модального окна
- Отображение модального окна после отправки формы

**Результаты:**
- Успешных: 35 из 60 тестов
- Основные проблемы: несоответствие селекторов, проблемы с фокусом

### 3. Формы и валидация (`forms.test.js`)

**Общее описание:** Тестирование форм обратной связи, включая валидацию полей, маски ввода и поведение при отправке.

**Ключевые проверки:**
- Наличие обязательных полей в формах
- Валидация телефона (корректный/некорректный номер)
- Наличие маски ввода для поля телефона
- Блокировка кнопки во время отправки
- Отображение ошибок при пустой отправке
- Успешная отправка формы с корректными данными

**Результаты:**
- Успешных: 40 из 75 тестов
- Основные проблемы: таймауты при ожидании загрузки, проблемы с валидацией

**Пример проблемного теста:**
```javascript
test('Валидация телефона: некорректный номер показывает ошибку', async ({ page }) => {
  // Тест падает из-за таймаута в beforeEach hook
  // Ошибка: Test timeout of 30000ms exceeded while running "beforeEach" hook
});
```

### 4. Адаптивность (`responsive.test.js`)

**Общее описание:** Проверка корректного отображения лендинга на различных устройствах и размерах экрана.

**Ключевые проверки:**
- Адаптация под мобильные устройства
- Корректное отображение на всех размерах экрана
- Работа на мобильных устройствах
- Отображение трансформаций на разных устройствах

**Результаты:**
- Успешных: 25 из 45 тестов
- Основные проблемы: различия в рендеринге между браузерами

## Анализ проблем и корневые причины

### 1. Таймауты при ожидании загрузки (42% упавших тестов)

**Симптомы:**
- `Test timeout of 30000ms exceeded while running "beforeEach" hook`
- `page.waitForLoadState: Test timeout of 30000ms exceeded`

**Корневые причины:**
- Слишком строгое ожидание `networkidle` в медленных окружениях
- Отсутствие fallback-механизмов при долгой загрузке ресурсов
- Проблемы с локальным сервером в некоторых браузерах

**Затронутые тесты:**
- Все категории, но особенно формы и навигация

### 2. Несоответствие селекторов (28% упавших тестов)

**Симптомы:**
- `Error: page.locator: Target closed`
- `Error: locator.click: Target detached`
- Элементы не найдены в DOM

**Корневые причины:**
- Динамическая генерация контента после загрузки страницы
- Изменение структуры DOM в разных браузерах
- Отсутствие стабильных data-атрибутов для тестирования

**Затронутые тесты:**
- Модальные окна, формы

### 3. Проблемы с валидацией форм (18% упавших тестов)

**Симптомы:**
- Ошибки валидации не отображаются
- Маска ввода телефона работает некорректно
- Кнопка отправки не блокируется

**Корневые причины:**
- JavaScript валидация может не срабатывать в тестовом окружении
- Асинхронная обработка событий ввода
- Различия в реализации между браузерами

### 4. Проблемы адаптивности (12% упавших тестов)

**Симптомы:**
- Элементы не отображаются на определенных размерах экрана
- Неправильное позиционирование в мобильной версии

**Корневые причины:**
- Медиа-запросы могут по-разному интерпретироваться эмуляторами
- Отсутствие тестов на реальных устройствах

## Рекомендации по исправлению

### 1. Улучшение стабильности тестов

**Для таймаутов:**
```javascript
// Вместо:
await page.waitForLoadState('networkidle');

// Использовать:
await page.waitForLoadState('domcontentloaded');
await page.waitForSelector('body', { state: 'visible' });
```

**Добавить retry-логику:**
```javascript
const retryOperation = async (operation, maxRetries = 3) => {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await operation();
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await page.waitForTimeout(1000);
    }
  }
};
```

### 2. Улучшение селекторов

**Добавить data-testid атрибуты:**
```html
<!-- В разметке: -->
<button data-testid="submit-form">Отправить</button>

<!-- В тестах: -->
await page.locator('[data-testid="submit-form"]').click();
```

**Использовать ролевые селекторы:**
```javascript
await page.getByRole('button', { name: 'Отправить' }).click();
await page.getByLabel('Имя').fill('Тест');
```

### 3. Оптимизация конфигурации Playwright

**Обновить `playwright.config.js`:**
```javascript
// Увеличить таймаут для CI
timeout: process.env.CI ? 60000 : 30000,

// Использовать более стабильное состояние загрузки
use: {
  // ... существующие настройки
  waitForLoadState: 'domcontentloaded',
},

// Добавить проекты для реальных мобильных тестов
{
  name: 'Real Mobile Chrome',
  use: { ...devices['Pixel 5'], isMobile: true },
}
```

### 4. Улучшение тестов форм

**Добавить явные ожидания:**
```javascript
test('Валидация телефона показывает ошибку', async ({ page }) => {
  await page.locator('input[type="tel"]').fill('123');
  await page.locator('button[type="submit"]').click();
  
  // Явно ждать появления ошибки
  await expect(page.locator('.error-message')).toBeVisible({ timeout: 5000 });
});
```

## Инструкции по запуску тестов

### Предварительные требования
- Node.js 18 или выше
- Установленные зависимости: `npm install`

### Базовые команды

**Установка Playwright браузеров:**
```bash
npx playwright install
```

**Запуск всех тестов:**
```bash
npx playwright test
```

**Запуск тестов с UI:**
```bash
npx playwright test --ui
```

**Запуск конкретного файла:**
```bash
npx playwright test tests/navigation.test.js
```

**Запуск тестов для конкретного браузера:**
```bash
npx playwright test --project=chromium
npx playwright test --project=firefox
npx playwright test --project=webkit
```

**Генерация отчета:**
```bash
npx playwright show-report
```

### Запуск в CI/CD окружении
```bash
# Установка зависимостей
npm ci

# Установка браузеров
npx playwright install --with-deps

# Запуск тестов
npx playwright test --reporter=html,line
```

### Отладка проблем
1. **Просмотр трассировки:** отчеты сохраняются в `playwright-report/`
2. **Скриншоты при падении:** доступны в `test-results/`
3. **Логирование:** использовать `--debug` флаг
4. **Интерактивный режим:** `npx playwright test --debug`

## Глоссарий

**Playwright:** Фреймворк для автоматизации тестирования веб-приложений, поддерживающий несколько браузеров.

**Селектор:** Выражение, используемое для поиска элементов DOM на странице.

**Таймаут:** Максимальное время ожидания выполнения операции перед тем, как тест будет считаться упавшим.

**Кросс-браузерное тестирование:** Тестирование приложения в разных браузерах для обеспечения совместимости.

**Адаптивное тестирование:** Проверка корректного отображения и работы приложения на различных размерах экрана.

**Page Object Model (POM):** Паттерн проектирования, при котором каждый веб-страница представлена в виде класса, инкапсулирующего её элементы и поведение.

**CI/CD:** Continuous Integration / Continuous Deployment — практика автоматической сборки, тестирования и развертывания приложений.

**data-testid:** Специальный атрибут HTML, используемый для идентификации элементов в тестах.

**Ретри (retry):** Повторная попытка выполнения операции после неудачи.

**Трассировка (trace):** Запись последовательности действий и состояний во время выполнения теста для отладки.

## Приложения

### A. Структура тестовых файлов
```
tests/
├── navigation.test.js    # Тесты навигации (45 тестов)
├── modals.test.js        # Тесты модальных окон (60 тестов)

```

### B. Конфигурация Playwright
Основные настройки в `playwright.config.js`:
- 6 проектов (браузеров/устройств)
- HTML репортер + консольный вывод
- Автозапуск веб-сервера на порту 8080
- Параллельное выполнение тестов

### C. Примеры проблемных скриншотов
Скриншоты упавших тестов доступны в директории `test-results/`. Наиболее частые проблемы:
1. **Таймауты загрузки:** белый экран или частично загруженная страница
2. **Отсутствие элементов:** элементы форм не отображаются
3. **Проблемы валидации:** ошибки не показываются пользователю
