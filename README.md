# Статический Next.js лэндинг - МеталлПро

Этот проект представляет собой собранный статический Next.js лэндинг для компании "МеталлПро", специализирующейся на производстве металлоизделий.

## Структура проекта

- `index.html` - главная страница
- `404.html` - страница 404
- `_next/` - статические ресурсы Next.js
- `.nojekyll` - отключает обработку Jekyll на GitHub Pages
- Конфигурационные файлы для различных хостингов

## 📚 Документация по деплою

Проект включает полную документацию для развертывания на различных платформах:

- **[DEPLOYMENT.md](./DEPLOYMENT.md)** - подробное руководство по деплою на GitHub Pages, Vercel, Netlify, общий хостинг и Cloudflare Pages
- **[CHEATSHEET.md](./CHEATSHEET.md)** - шпаргалка с быстрыми командами для экспресс-деплоя
- **Чек-лист преддеплойной проверки** - включен в DEPLOYMENT.md
- **Инструкции по мониторингу** - как проверить работоспособность после деплоя

Рекомендуем начать с [DEPLOYMENT.md](./DEPLOYMENT.md) для пошаговых инструкций.

## Конфигурационные файлы

Проект включает готовые конфигурации для развертывания на популярных платформах:

### 1. GitHub Pages
- **CNAME** - файл для настройки кастомного домена (заполнить при необходимости)
- **.nojekyll** - уже присутствует, отключает обработку Jekyll
- **Рекомендации**: 
  - Запушить проект в репозиторий GitHub
  - В настройках репозитория выбрать ветку и папку `/(root)`
  - При использовании кастомного домена указать его в CNAME

### 2. Vercel
- **vercel.json** - полная конфигурация для Vercel
- **Автоматическое развертывание**:
  ```bash
  # Установите Vercel CLI
  npm i -g vercel
  
  # Деплой
  vercel --prod
  ```
- Или подключите репозиторий через веб-интерфейс Vercel

### 3. Netlify
- **netlify.toml** - конфигурация для Netlify
- **_headers** - security headers
- **_redirects** - правила перенаправления
- **Деплой**:
  - Перетащите папку проекта в Netlify Drop
  - Или используйте `netlify deploy --prod`

### 4. Общий статический хостинг (Apache/Nginx)
- **.htaccess** - конфигурация для Apache
- **nginx.conf** - базовая конфигурация для Nginx
- **Инструкции**:
  - Скопируйте все файлы в корневую директорию хоста
  - Для Apache: убедитесь, что включен mod_rewrite
  - Для Nginx: используйте предоставленный конфиг как основу

## Security Headers

Проект включает настройки безопасности:
- `_headers` - для Netlify/Cloudflare Pages
- Заголовки настроены в `.htaccess` и `nginx.conf`
- Включены: X-Frame-Options, X-Content-Type-Options, CSP, HSTS (пример)

## SPA Роутинг

Для корректной работы Single Page Application настроены:
- Все маршруты ведут на `index.html` (кроме статических файлов)
- Файл `_redirects` для Netlify
- Правила в `vercel.json` и `.htaccess`

## SEO Оптимизация

- **sitemap.xml** - карта сайта (настроить URL под домен)
- **robots.txt** - уже присутствует в проекте
- **humans.txt** - информация о разработчиках
- Мета-теги настроены в `index.html`

## Локальная разработка

Для локального просмотра можно использовать любой статический сервер:

```bash
# Python 3
python -m http.server 8000

# Node.js
npx serve .

# PHP
php -S localhost:8000
```

## Настройка домена

1. **GitHub Pages**: укажите домен в CNAME файле
2. **Vercel**: настройки домена в Dashboard → Domains
3. **Netlify**: Domain settings → Add custom domain
4. **Свой хостинг**: настройте DNS записи (A/AAAA или CNAME)
