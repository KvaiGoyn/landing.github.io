# Техническая спецификация: Реализация функционала кнопок для проекта "МеталлПро"

## Обзор
В проекте Next.js "МеталлПро" обнаружено 16 уникальных кнопок в 7 компонентах. Все кнопки имеют только визуальное оформление, обработчики событий не реализованы. Данная спецификация описывает необходимый функционал для каждой кнопки, архитектуру реализации и приоритеты.

## Архитектура реализации

### Общие утилиты/хуки
1. **`useScrollTo`** - хук для плавного скролла к элементам по ID
   ```typescript
   const scrollToSection = (sectionId: string) => {
     const element = document.getElementById(sectionId);
     if (element) element.scrollIntoView({ behavior: 'smooth' });
   };
   ```

2. **`useModal`** - хук для управления состоянием модальных окон
   ```typescript
   const { isOpen, openModal, closeModal, modalType, modalData } = useModal();
   ```

3. **`useForm`** - хук для управления состоянием форм (валидация, отправка)
   ```typescript
   const { register, handleSubmit, errors, isSubmitting, submitForm } = useForm();
   ```

4. **`useCarousel`** - хук для управления каруселью отзывов
   ```typescript
   const { currentSlide, totalSlides, nextSlide, prevSlide, goToSlide } = useCarousel(items);
   ```

5. **`useAccordion`** - хук для управления аккордеоном FAQ
   ```typescript
   const { expandedId, toggleAccordion } = useAccordion();
   ```

### Состояние приложения
- **Локальное состояние**: Для простых компонентов (аккордеон, карусель) используем `useState`
- **Глобальное состояние**: Для модальных окон и данных форм используем React Context (`AppContext`)
- **Схема состояния**:
  ```typescript
  interface AppState {
    modals: {
      isOpen: boolean;
      type: 'calculation' | 'callback' | 'consultation' | 'order' | 'measurement' | 'projectDetails' | null;
      data: Record<string, any>;
    };
    forms: {
      [key: string]: FormState;
    };
    portfolio: {
      activeFilter: string;
    };
  }
  ```

### Дополнительные компоненты
1. **`Modal`** - общий компонент модального окна
   - Props: `isOpen`, `onClose`, `title`, `children`
   - Анимация появления/исчезновения
   - Оверлей, закрытие по клику вне или ESC

2. **`FormModal`** - специализированный модальный компонент для форм
   - Наследует от `Modal`
   - Добавляет форму, кнопки отправки, состояния загрузки

3. **Конкретные формы**:
   - `ContactForm` - форма в разделе Contact
   - `CallbackForm` - форма обратного звонка
   - `CalculationForm` - форма расчета стоимости
   - `ConsultationForm` - форма консультации
   - `OrderForm` - форма заказа услуги (с параметром service)
   - `MeasurementForm` - форма вызова замерщика
   - `ProjectDetails` - детали проекта (не форма)

4. **`Accordion`** - компонент аккордеона
   - Управляемое/неуправляемое состояние
   - Анимация раскрытия

5. **`Carousel`** - компонент карусели
   - Бесконечная прокрутка
   - Адаптивность
   - Пагинация

### Интеграция с бэкендом
1. **API эндпоинты** (пример):
   - `POST /api/contact` - контактная форма
   - `POST /api/callback` - обратный звонок
   - `POST /api/calculation` - расчет стоимости
   - `POST /api/order` - заказ услуги

2. **Сервисный слой** `src/services/api.ts`:
   ```typescript
   export const api = {
     sendContactForm: (data: ContactFormData) => axios.post('/api/contact', data),
     sendCallbackRequest: (data: CallbackData) => axios.post('/api/callback', data),
     // ...
   };
   ```

3. **Обработка ошибок и успеха**:
   - Показ уведомлений (toast)
   - Сброс формы после успешной отправки
   - Блокировка кнопки во время отправки

## Технические требования

### Типизация
- Использовать TypeScript для всех новых компонентов и хуков
- Определить интерфейсы для пропсов, состояний, данных форм

### Стилизация
- Сохранить существующие классы Tailwind CSS
- Добавить состояния через conditional classes
- Анимации через Tailwind `transition` и `transform`

### Доступность (a11y)
- Кнопки должны иметь `aria-label` где необходимо
- Модальные окна должны управлять фокусом
- Аккордеон должен иметь `aria-expanded`

### Производительность
- Ленивая загрузка модальных окон (dynamic import)
- Мемоизация обработчиков с помощью `useCallback`
- Оптимизация ререндеров с `React.memo`
